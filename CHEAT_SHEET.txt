================================================================================
PROTOCOLANALYZER CHEAT SHEET
================================================================================

QUICK START
-----------
1. Clone: git clone https://github.com/DonkRonk17/ProtocolAnalyzer.git
2. Run: python protocolanalyzer.py analyze ./your-project
3. Compare: python protocolanalyzer.py compare websocket socket.io

No dependencies required - Python 3.7+ only!

================================================================================
CLI COMMANDS
--------------------------------------------------------------------------------

ANALYZE PROJECT:
  protocolanalyzer analyze <path>                    Basic analysis
  protocolanalyzer analyze <path> -r realtime        Specify requirement
  protocolanalyzer analyze <path> -o report.md       Output to file
  protocolanalyzer analyze <path> -f markdown        Format: json|markdown|text
  protocolanalyzer analyze <path> -v                 Verbose mode

COMPARE PROTOCOLS:
  protocolanalyzer compare websocket socket.io       Compare 2+ protocols
  protocolanalyzer compare ws http grpc -f json      Multiple with format

MIGRATION GUIDE:
  protocolanalyzer migrate socket.io websocket       Get migration steps
  protocolanalyzer migrate grpc http_rest -f md      Format as Markdown

LIST PROTOCOLS:
  protocolanalyzer list                              Show all protocols
  protocolanalyzer list -c realtime                  Filter by category

================================================================================
REQUIREMENT TYPES
--------------------------------------------------------------------------------

  realtime            WebSocket, Socket.IO, MQTT (bidirectional)
  request-response    HTTP/REST, GraphQL (client-initiated)
  streaming           SSE (server push)
  rpc                 gRPC (remote procedure calls)
  auto                Auto-detect from codebase

================================================================================
PYTHON API
--------------------------------------------------------------------------------

BASIC USAGE:
  from protocolanalyzer import ProtocolAnalyzer
  
  analyzer = ProtocolAnalyzer()
  result = analyzer.analyze("./project")
  
  print(result.architecture_type)
  print(result.detected_protocols)
  print(result.recommendations)

COMPARE PROTOCOLS:
  comparison = analyzer.compare_protocols(["websocket", "socket.io"])
  for name, data in comparison.items():
      print(f"{name}: {data['complexity']}/10")

MIGRATION GUIDE:
  guide = analyzer.get_migration_guide("socket.io", "websocket")
  print(guide['difficulty'])
  for step in guide['steps']:
      print(step)

EXPORT REPORTS:
  json_output = analyzer.to_json(result)
  markdown_output = analyzer.to_markdown(result)

VERBOSE MODE:
  analyzer = ProtocolAnalyzer(verbose=True)

================================================================================
COMPLEXITY SCORING
--------------------------------------------------------------------------------

Score = base_complexity * scale * confidence * spread

Factors:
  - Base complexity: Protocol inherent difficulty (1-10)
  - Scale factor: Lines of code usage (1.0-2.0)
  - Confidence: Detection pattern confidence (0.5-1.0)
  - Spread factor: Number of files (1.0-1.5)

Final score: 0-100 (higher = more complex)

================================================================================
RECOMMENDATION SCORING
--------------------------------------------------------------------------------

Bonuses:
  +20 points    Matches your requirement category
  +25 points    Already in use (consistency)
  +10 points    Same category as existing protocols
  +10 points    Simple protocol (complexity <= 3)

Penalties:
  -27 points    High complexity (max penalty)

--------------------------------------------------------------------------------

    → Socket.IO v4 client requires v4 server

[!] Multiple real-time protocols detected
    → Consider consolidating for simplicity

[!] High complexity protocols detected
    → Review if simpler alternatives exist

================================================================================
TROUBLESHOOTING
--------------------------------------------------------------------------------

No protocols detected?
  → Check file extensions are in scan list
  → Verify not scanning inside node_modules
  → Use --verbose for debugging

False positives?
  → Check confidence scores in JSON output
  → High confidence (0.9+) is typically accurate

Wrong recommendation?
  → Specify --requirement explicitly
  → Check if your use case matches the category

Migration time seems wrong?
  → Use difficulty (LOW/MEDIUM/HIGH) as primary indicator
  → Estimates based on file count
